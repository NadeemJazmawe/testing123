Description

The purpose of the document is to explain how to work with Ginger Variable Dependency.

Add Variable Dependency to Business Flow

There are two types of Variable Dependency.

Add Activity Variable Dependency

To Activate Variable Dependency, Click on the Activities-Variable Dependencies.

In the pop window, enable the checkbox Activities-Variable Dependencies Control. Click OK

Then click on Business Flow Variable and add one selection list variable.

Open Edit page of variable selection list and add optional values in that list.

Repeat Step-a and open the Activities-Variables Dependencies page. Now we can see the new extra columns of variable selection list values

followed by activities.

Here for variable selection list: a, selected activities are 1st and 2nd which means for variable value a, only those activities get executed. Like that,

for variable selection list: b, selected activities are 1st and 3rd which means for variable value b, only those activities get executed.

By default variable selection list value is first means here selected value: a, so only 1st and 2nd activities gets executed and remaining are marked

as skipped.

Let’s move towards the execution, click on Run Flow you can see execution like below.

Now open the edit page of the variable list and change the selected value: b.

Click on Run Flow and you can see only 1st and last activities get executed for value b.

Add Action Variable Dependency

To Activate Variable Dependency, go to automate page and Click on the Actions-Variable Dependencies.

In the pop window, enable checkbox Actions-Variables Dependencies Control. Click OK

Then click on Activity Variable and add one selection list variable.

Open Edit page of variable selection list and add optional values in that list.

Repeat Step-a and open the Actions-Variables Dependencies page. Now we can see the new extra columns of variable selection list values followed by actions.

        f.    Here for variable selection list: Chrome, selected actions are 1st and 2nd which means for variable value Chrome, only those actions get

         executed. Like that, for variable selection list: Firefox, selected activities are 3rd and 4th which means for variable value Firefox, only those             actions gets executed.

     g.   By default variable selection list value is first means here selected value: Chrome, so only 1st and 2nd actions get executed and remaining          are marked as skipped.

        Let’s move towards the execution, click on Run Activity you can see activity execution like below.

    h.   Now open the edit page of the variable list and change the selected value: Firefox.

      i.  Click on Run Activity and you can see only 3rd and 4th actions get executed for value Firefox.
