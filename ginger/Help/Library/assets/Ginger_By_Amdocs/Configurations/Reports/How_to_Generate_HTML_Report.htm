         Run Set Configuration on the Run tab

             Go to the “Run Sets” tab, select your “Run set”.

Click on the “Details” tab.

 

Here user can update the name of the Run set, description and select Publish check box.

If you want Analyzer is required before manual execution of the Run set then go to the Runners and Click on the Runners Execution Configuration icon as shown below;

 

Select Analyze Runners Before Execution

 

Producing Reports of the execution

Here user can add various activities that will be done before/after running set execution.
‎One such activity is “Add Produce HTML Report Operation”.

Go to the Operations tab and click on the relevant button. Users can add as many HTML reports for one execution.

 

By adding this action, the report will be created after the completion of execution.
‎Used template of the report can be changed from Select Report Template. By default, the selection is the “Default” report.

The folder for saving the current report can be changed from ‘Use custom root path for report folder.’ Click on the check box of the ‘Use custom root path for report folder’ option and select desired folder.

 

Once such action is added then a report will be created per each execution, with options described in this action.
‎

Sending Reports via email

Users can send reports via email before/after running set execution.‎

Go to the Operations tab and click on ‘Add Send HTML Report Email Operation.’

 

Enter the desired email ids in the From and To fields, subject, and select the Email Body Report template of the email.

 

 

Go to the attachments tab and click on the ‘Add Report’ button. This is will send the HTML Reports as an attachment in the mail.

Run button is displayed at the bottom to check if the test mail is delivered successfully.

 

Click on the Save button to save the updates to the Run set.

 

 

            Run Set list

 

            Presentation of all Run Set executions (that are done at the “Run Sets” tab) is available as the list in the Executions History tab.

Reports per execution also can be produced. Generate Reports can be done instantly and on-demand.

For customization of such a report will automatically be used as a default template.

 

Search, Refresh execution results, Open Execution Results Main Folder, Delete Selected Execution Results, Delete All Execution Results these functionalities are available in this window.

To generate a report for the specific run set, click on the “Generate Report” column at the specific row.


‎

The folder with the new-created report will be opened automatically.

 

 

           Report Options

            It is possible to create the report for the last execution that was done via the “Automation” tab.

 

For this - run one from 2 possible ways to do the execution – “Run Action”, “Run Activity” or “Run Flow”.

Once it’s done - you able to process the report – click on “Generate Report” -> “Generate Last Executed Item Report” or “Generate Full Business Flow Report”

              

 

              In the Report Option, we can generate 3 types of reports.

  •     Open Business Flow Execution Summary Page
  •     Produce Business Flow Execution Report
  •     Auto Produce business Flow Execution Report after Flow Execution

 

    Open Business Flow Execution Summary Page

             To produce this report, click on the Run Flow after completion of the execution of the flow the following report will get a display.

            

 

           You can click on the Generate Report to generate Ginger Execution Report.

            

 

 

   Produce Business Flow Execution Report

            To produce this report, click on the Run Flow after completion of the execution of the flow click on the Produce Business Flow           

            Execution Report from the Report Option.

           

 

           After clicking on the above option Ginger Execution Report will get generate.
