Introduction

The purpose of this document is to provide a hands-on step by step user guide for Autopilot Page Objects Model feature and to provide a wide overview on all Page         Objects Model available functions.

It is available for Web and Java Platforms.

 

Page Objects Model Wizard

      There are 2 ways to create a Page Objects Model, first is to create one manually and the second is to let the Ginger learn the Page you’d like to automate, both creations are being done using the Page Objects Model Wizard.

 

Automatic page learn

Prerequisites:

Solution with at least one Platform Target Application

At least one platform agent (e.g. selenium chrome Agent for Web Platform)

Site URL which you wish to learn.

 

Go to Resources tab à Applications Models à Page Objects Models.

Click the Plus button.

 

Add Page Objects Model Wizard is opened.

Click Next Introduction page.

Learning Configurations page:

Select Target application ---for Web , Mobile, Window and Java

(If you don’t have any Web Platform Target Application in your solution, please close the wizard configure a new target application configure

one agent, and start over from step 1).

 

For Java Applications:-

In order to use the Java agent in the Page Object Model screen, the Java agent needs to be attached to the Java application from the Business Flow tab as shown below.

 

Run the Launch Java Application Action and attach the Java agent to the Java application.

 

Either launch the application using this Action or attach the agent on already launched Java application.

 

 

In the Add POM wizard, select the Target application as MyJavaApp. The Java agent is automatically populated and is Green indicating that it is On.

 

Learn Specific Frame (Java Driver):

Checked Learn Specific Frame, in order to learn specific frame of the selected window.

 

Select Agent (If you don’t have an Agent in your solution, please close the wizard configure a new Selenium agent, and start over from step 1).

 

After selecting the agent, the On/Off button will be enabled. Click it in order to turn on your agent.

 

Now go to the browser instance which been raised by Ginger and navigate to the page URL you would like to learn.

 

When the Agent and its browser instance are opened, the list of Elements Types will be shown on the page.

 

Here you can specify what types of element interesting you in order to create your automation.

Once done selecting all desired elements click next.

 

Custom Relative Xpath (Supported Web and Mobile) :

On Checked of Custom Xpath Learning Configuration, Custom xpath grid will be visible.

You can add multiple XPath format (combine multiple attributes as shown in below SS) in list. During POM learning Ginger will create relative XPath for element,

if it match the format.

For Example : if you want to create relative xpath of below web element with combination of “placeholder” and “type” attributes. Then xpath format will be //*

[@placeholder=’ ‘and @type = ‘ ’].

During Learing, ginger will look for attributes mentioned in format for each web element and for matching web elements , it will replace single Quotes (‘ ’) with

its original value.

For above format relative xpath will be “//*[@placeholder=’First Name’ and @type=’text’]”.

              

 

             

 

Learned Object Mapping

When reaching to this page the learning process will be automatically started and the Ginger will start to learn all existing elements on the page you were navigated on the previous steps.

Indication of the running process would be the spinning orange icon which becomes visible. Also, you will notice that the elements are being added to the mapped elements and unmapped elements grids.

The mapping between them is done depending upon the types of elements you selected on the learning configuration page.

 

Selecting any element on the grid will cause the details section to be opened and there you can find all the learned properties and locators of the selected element.

 

If you wish to stop the learning process (reason can be Ginger already learned all relevant elements and there is no reason to continue or you simply want to change the page on the browser instance etc.) you can do it by clicking the stop button.

 

After clicking the stop button, you will be able to restart the learning process by clicking Re-learn button

 

Using the Spy:

Click on the Spy button

 

While the Spy is on move your mouse to the requested element on the page and click ctrl.

  • Element not found – re-try 2 or 3 times.
  • An element found – the element will be focused on the elements grid
  • An element found but does not exist on the Element Grid – you will get a button to add the found element to the grid.

 

Click Next.

              General Details page:

The general details page contains all details which been pulled for the page, here you

can change the page name, URL, and screenshot.

 

Click Finish – The Page Objects Model has been added to the Page Objects Model tree.

 

      Manual Page Objects Model creation

 

Go to Resources tab à Application Models à Page Objects Models à and Click the Plus button.

            

 

Add Page Objects Model Wizard is opened.

Click Next Introduction page.

Check the Manual Radio Button and click Next.

 

Learned Object Mapping:

Add Elements By clicking the Plus button

 

Add Locators to your added elements

 

General Details

Add Name, Description (optional) and Page URL (optional)

 

Browse and Image for your Page Objects Model (optional)

Click Finish, the New Page Objects Model added to the tree.

 

        Adding Unmapped Elements to Mapped Elements list and vice-versa

 

            If an element is an unmapped list and the user desires to use it in the Actions then follow the below

             steps.

  •             Go to the Unmapped Elements tab and select the element.
  •             Click on the ‘Add Elements to mapped list’ button as shown below

 

 The element will move to the Mapped Elements tab as shown below

 

The same steps are to be performed while moving the elements from mapped to unmapped tab.

 

 

      Learn Only Mapped Elements

             Users can choose an option to learn only mapped elements instead of learning all the elements of the page.

             This reduces the learning time. This option is available by default.

 

            The user has to uncheck the checkbox if unmapped elements are to be learned.

           

 

 

      Learning the Elements by Locator’s Priority

            Users can set the priority of locators of the elements of the page.

            On the Learning Configurations page, expand the area ‘Learned Elements Locators Preferences’.

           There are 4 options viz. ById, ByName, ByRelXPath & ByXpath.

            The sequence of the priority can be changed by using the arrow buttons as indicated below.

            In this screenshot, when any element will be used in an Action, the Locator used will be ‘ByID’.

            

 

 

            Users can also change the Locator of a particular element after the page is learned.

            For this, select the Page Object Model. Go to the UI Elements tab and select the desired element.

            Expand the Details area. Select the desired locator and using the arrow buttons bring it to priority 1.

             

 

 

Giving Dynamic value to a Locator of the Learned Element

 

              A locator of a learned element can be given a customized value.

             For this, select the Page Object Model and go to the UI Elements tab.

             Select the desired Element and expand the Details area.

             Go to the Locators tab and select the desired locator.

             Copy-paste the locator.

             The pasted locator can then be edited by either directly typing in the value field or by using the

              value expression button ‘…’ as shown below.

              

 

 

Page Objects Model Edit Page

 

Prerequisites:

At least one Page Objects Model was created on the Page Objects Model tree.

 

Page Objects Model General Details

Go to Resources tab à Application Models à Page Objects Models.

Select the Page Objects Model on the tree.

 

On the upper section of the screen, you can change the Page Objects Model name description and the Target Application.

Select an Agent to work with and turn it on by clicking the Red button, if the selected Agent is already raised, the button will be automatically green,

and no need to re-raise the Agent.

 

By default, the agent will be opened with the specified Page Objects Model Page URL, whenever you need you can navigate to this URL by clicking the Arrow button.

 

 

Screenshot Tab

                Changing the Screenshot can be done by Clicking the Take Screenshot button which will get a screenshot from the current page on the mapped

               Agent or by browsing Image from a local file.

              

 

UI Elements Tab

The page is divided into two :

Mapped elements – The elements which we will use to automate our page

Unmapped elements – Repository of all unused elements on the page.

 

        Clicking the signs icon will shift elements from one group to another

 

       

 

 

  Select any element will enable the properties and locators grids in the below section

   

 

 

You can test a specific locator by clicking its play button or click the grid’s menu play button to check all locators, the locator

statuses will be changed according to the test result.

 

 

You can test all elements by clicking the Test All Mapped Elements button, this operation will approve your flow will not going to fail because the element cannot be found on the page, here you will have the opportunity to be aware that an element cannot be found before your execution and fix it.

If you switch to the unmapped element tab you the button will be changed to test all Unmapped elements.

 

You can add a new locator by clicking the add button, or copy and paste an existing one and then modify it for your needs.

The Up and Down buttons will define the priority of the locators, the priority order will be taken into consideration while searching the locator of the page.

If you would like to exclude locators from the list just Uncheck its Active field.

Note: In automating section explains that in run time all Active locators will be used till the element will be found.

You can highlight elements on the open instance of the Agent by clicking the Highlight button.

 

Using the Spy:

Click on the Spy button

 

While the Spy is on move your mouse to the requested element on the page and click ctrl.

Element not found – re-try 2 or 3 times.

An element found – the element will be focused on the elements grid

An element found but does not exist on the Element Grid – you will get a button to add the found element to the grid.

 

 

Notes:

Most of the operations (Highlight, Test Elements/Locators, Spy, Go to Page URL, etc.) will work only if the Agent is Selected and turned on.

Auto Learned elements cannot be modified or deleted as they take from the page and in order to maintain the page we should keep them as is, manually added elements can be modified and deleted to your need.

 

Update Elements

                If the elements of the learned page have been updated then the user can update the elements using the ‘Update Elements’ button

1. Select the Page Object Model and go to the UI Elements tab.

2. Turn On the Agent and go to the desired page in the agent.

3. Click on the ‘Update Elements’ button

 

 

4. ‘POM Elements Wizard’ is displayed.

Expand the ‘New Elements Learning Settings’ and the ‘Learned Elements Locators Preferences’

areas to get the details of them.

 

Users can make changes in these areas if desired.

Click on the Next button.

 

5. All the elements present on the page are compared with the latest status.

In order to map the element which is new and cannot be mapped, Press “Next”

 

               6. A new window “Map deleted element with the new element” is displayed. If there is no deleted element found on the page, then this grid will

               be empty.

Select the edit option to map the element with new elements found on the page.

 

 

7. Once the new element got selected, the user can validate the properties and values of existing/deleted elements and newly added elements

by selecting the “Compare” option.

Also, the properties and locators of a deleted element will be replaced with new values and will avoid breaking the automation flow

which was happening earlier.

 

 

 

  The updated elements will be displayed on the Elements Compare page.

     The Comparison Status and Comparison Details fields will give the details of it.

     As shown below, a green check means ‘Unchanged’. The description of the icon will be displayed in the tooltip when the mouse hovers over the

icon.

 

 

Page Objects Model Element’s Values

             After the page is learned, select the Page Objects Model on the tree.

             Go to the UI Elements tab.

            As shown in the below screen, the Possible Values field is displayed.

            The values are also populated from the page if any.

           As shown in the below screenshot, the selected element has learned the possible values from the page.

            

 

              Elements that can have values are only allowed eg. Text box, list, etc.

             For all the other elements, a warning message is displayed

              

 

             To enter the values, click on the ‘…’ button near the possible values field on the require element

            

 

 

            Click on the + button in the new window

           

 

           User can add multiple values here and set one of them as default and click on the OK button.

 

           User can also modify and delete the values from this window.

             

 

                The default value will be displayed with a * in the list

              

 

 

 

       Using Page Objects Model for Automate and Execute

          Automate Using Page Objects Model

                   Go to Business Flows Tab à Select the desired Business Flow with Web/Java Platform as Target Application.

Double click on the Business Flow.

Select the desired Activity where you want to add the Action.

Expand the Add Actions section

 

 

        Select UIElement Action and click on Add to Actions button to add it.

           

 

             Double click on the UI element action to open it in Edit mode.

 

              Select “Page Object Model Element” as Locate By. Select your added Page Objects Model in the “Page Object Model”.

 

              Select your desired element as “Model Element” and click on the ‘Select Element’ button.

 

Select the Element Action from the list and populate the value if needed.

 

 

              Click on Run Action.
